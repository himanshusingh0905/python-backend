Ask yourself this one question:
"Do I need to produce or process a sequence of items?"
↓ yes
use a generator
Output comes one item at a time. You process each item and move on. You don't need everything at once.
↓ no
just a regular function
You need one single result. Compute it and return it. No sequence involved.
Then ask — is the data large or infinite?
large / infinite / unknown
generator is essential — don't load it all into RAM
small / known size
generator still works, but a list is fine too
Real signals in your code that scream "use a generator"
"I'm reading a big file line by line"
"I'm fetching pages from an API"
"I'm processing millions of DB rows"
"I need to insert data in batches"
"I'm building a data pipeline"
"I need an infinite sequence"
